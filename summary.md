To approach the problem, I first focused on understanding what the data represents and what information is available at the time a new customer is assigned. Based on the read me, the key inputs are communication method, lead source, destination, and launch location. My goal was to use each of these variables to match the incoming lead with the agent who has performed best in similar situations historically. I used Visual Studio Code to write and organize the SQL files, and DBeaver to inspect the database schema, explore the data, and execute/test my queries

I started by exploring the data to understand the relationships across the tables. I specifically looked at booking status and noticed that outcomes can be Confirmed, Cancelled, or Pending. Since Pending does not represent a final outcome, I excluded it from agent performance, using only Confirmed and Cancelled bookings. This allowed me to calculate a reliable close rate for each agent.

Next I built a baseline metric for each agent by calculating their overall close rate and average confirmed revenue. The overall close rate represents how effective an agent is across all scenarios and serves as a independent variable.

I then introduced more defined logic by evaluating agent performance across each assignment time attribute like communication method, lead source, destination, and launch location. For each of these, I calculated a context specific close rate for each agent.

However, I wanted to avoid overfitting to small sample sizes. Some agents may have little or no history in certain categories, which could produce inaccurate results. To address this issue I implemented a fallback rule that if any agent has fewer than two decided records in a given category, I use their overall close rate instead of the context specific rate. This ensures the model remains stable and does not overweight limited data.

Lastly, I combined each component into a single ranking score. I weighted base_metrics.close_rate at 70% to reflect each agent’s overall historical performance, used the average of method_history.close_rate, source_history.close_rate, destination_history.close_rate, and launch_history.close_rate at 20% to capture how well the agent matched the incoming lead, and included space_travel_agents.AverageCustomerServiceRating/5.0 at 10% as an added quality check. The final result is a ranking model that accurately assigns a incoming customer to an agent most likely to make a sale and provide a great customer service experience.
